Using Web APIs

What Are APIs?

An API, or "Application Programming Interface," is simply a set of rules for communication with an application. For example, an application may be designed to serve data from a database, and have an API which a programmer can use to ask for specific data. This makes it possible for different applications to communicate with each other.

We'll primarily be focused on learning how to use publically accessible APIs on our websites. Web APIs are APIs that are accessed via the internet, usually using the HTTP(S) protocol, the same protocol used to access websites.

Using Web APIs

Since web APIs often use HTTP, we can access them by making HTTP requests in the form of a URL. To do this in our JavaScript code, we can use the Fetch API, a feature provided by JavaScript.

However, there's a complication when using the Fetch API, which is that it is asynchronous in nature. This means that it operates outside of the regular flow of our script. This is because making a remote request can take time, which, if synchronous, would tie up our code while we wait for a response. So, in order to use the Fetch API, we'll need to first learn how to write asynchronous code.

Async and Await

When we call an asynchronous function, like fetch, we don't know when it will finish getting a response. To address this, asynchronous functions return a Promise of a result. A Promise is a placeholder which can eventually be used to access the actual response value. The await keyword tells part of our code that it must wait for a Promise to be resolved. We usually put our asynchronous code inside of a function with the async keyword before it

async function getData() {
    const response = await fetch("https://emojihub.yurace.pro/api/random");
    const data = await response.json();

    document.querySelector(".emoji-display").innerHTML = data.htmlCode[0];
}

Above we have an example of an API call to an emoji API. We use async and await to get data from the API and then display some of that data onto our webpage.
