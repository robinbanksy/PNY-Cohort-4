# HTML Layout

The purpose of HTML is to provide a structured way to display content. We've already seen how heading tags (h1-h6) can be used to organize and structure the content on a webpage. In general, when writing HTML for our webpages, we should be thinking in terms of **layout.**

What this means, is that we should be able to look at a sketch or model of the page we are building and quickly break down the layout into distinct sections.

Once we have an understanding of the desired layout, we can use **semantic** HTML elements to create a well-structured and accessible webpage.

## What are Semantic HTML Elements?

Semantic HTML elements are those that clearly describe their meaning in a human-readable way. They provide context about the content they contain, making the HTML more accessible and easier to understand.

Some common semantic elements include `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<aside>`, and `<footer>`.

All of these tags are essentially empty boxes; they don't do anything visually on their own. Rather, they are used to define the structure and meaning of the content they contain. For example, the `<nav>` tag is used to define a navigation menu. We don't need to use a nav tag to create a navigation menu, but it makes our HTML more semantic and easier to understand.
