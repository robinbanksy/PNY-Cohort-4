What is Git?

Git is a kind of software known as a version control system. Essentially, it allows us to keep track of changes that we make to our projects. This is useful, because it lets us revert to previous versions if needed, and also makes it easier to collaborate with developers.

To use git, we create a Git repository, a folder in which we will store any project files. Git functions within the repository folder, providing its version control features.

What is GitHub?

Github is an online host for Git Repositories. Instead of storing our code locally, we can have our Git repositories hosted on Github, so that they are accessible from anywhere.

The copy of our code stored on GitHub is called a "remote" repository. The copy on our device is called a "local" repository. We use Git commands to sync changes between the two. The usual workflow is the following:

  1. Pull changes from the remote repository on GitHub to our local repository.
  2. Make changes to our code in the local repository.
  3. Use Git commands to push those changes back to the remote repository on GitHub.

Github Codespaces

In the previous class, we were introduced to the code editor VS Code. Github Codespaces allows us to use a version of VS Code directly in our browser, without having to install it locally.

Codespaces are created on a particular branch of a repository, and stand in as our local repository.

HTML Attributes

HTML attributes provide additional information about an element. They are specified inside of the opening tag of an element and consist of a name and a value. We've already seen some examples of HTML attributes in action, such as the "href" attribute in the anchor tag.

Here are some examples of attributes:

<h2 id="section-one-heading">Section Title</h2>
<a href="https://example.com" target="_blank">Link</a>

HTML IDs

As seen above, an id is a kind of HTML attribute. We use ids to uniquely identify elements, thus allowing them to be targeted with CSS or JavaScript. Two elements on the same page cannot have the same id.

The Anchor Tag

The anchor tag (<a>) is used to create hyperlinks in HTML. It allows users to navigate to other pages or sections within the same page.

In the previous class, we briefly saw that the anchor tag can link to external websites. Another use for them is to link to other pages on our own site. We accomplish this by pointing the href attribute to another HTML file in our site's folder:

<a href="about.html">About Us</a>

We can also use anchors to link to specific sections within the same page, done by instead pointing the href to an id on the same page:

<a href="#section-one">Go to Section One</a>

The Image Tag

We no doubt want to include images in our web pages. The image tag (<img>) is used to display images in HTML. Notice that it is an empty tag, meaning it does not have a closing tag.

To use the image tag, we need to specify the source of the image using the "src" attribute. This can be a URL to an image hosted online, or a path to an image file in our project folder:

<img src="https://example.com/image.jpg" alt="Description of image">

It is also important to include the "alt" attribute, which provides a text description of the image for accessibility purposes. This description will be read by screen readers for visually impaired users, and will also be displayed if the image fails to load.
