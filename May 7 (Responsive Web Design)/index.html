Responsive Web Design

When we design webpages, we are almost always doing it on a desktop or laptop computer. However, in the modern day, it's very possible that the majority of our users will instead be accessing our websites on mobile devices.

Unsurprisingly, the difference in screen size and orientation can significantly impact how our content is displayed and interacted with. To address this, we make use of the concept of responsive design.

The idea behind responsive design is to create webpages that automatically adjust their layout and content to best accomodate the properties of the client device. In the most extreme cases, this could mean that a webpage has one layout for desktops and a completely different one for mobile devices.

Key Principles of Responsive Design

So, how do we apply responsive design in our projects? The answer can vary depending on the specifics of each webpage, but we can define our approach based on the following principles:

  1. Less in more: If you make a very simple HTML page with minimal styling, it's very likely that it will already be responsive; by default, text content in an HTML document will wrap to fit available space. What this means is that if we are having issues with responsivity, it's probably because of something we've added to our CSS styling.
  2. Use explicit dimensions sparingly: When we specify a fixed width or height to an element, we prevent that element from modifying its size to fit the page or its content. A common solution is to instead use max-width, max-height, min-width, and min-height. In some cases, we can also make use of relative units like % or em, which can change to match client device properties.
  3. Use flexible layouts: Instead of using fixed positioning, consider using CSS Flexbox to create layouts that can adapt to different screen sizes and orientations.
  4. Use media queries: If all else fails, we can use media queries to apply different styles based on the characteristics of the client device.

Meta Viewport Tag

As an aside, many mobile browsers operate in such a way that they assume the webpage is designed for a desktop screen. As a result, they often render the webpage at a full desktop size and then shrink it to fit the mobile screen size. This can result in content being reduced to an unreadable size. Thankfully, the fix is easy; we simple include the following metadata tag in the <head> section of our HTML documents:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Media Queries

Media queries are a feature in CSS that allow us to ask the client device about its characteristics, such as its width, height, and orientation. We can then apply different styles based on these characteristics.

Below is an example of a media query:

@media (max-width: 800px) {
    /* CSS rules for screens with a maximum width of 800px */
}

As seen above, the query begins with the flag @media, followed by a condition in parentheses. The condition specifies the characteristics of the client device for which the styles should be applied, in this case, screens with a maximum width of 800px. This means that we can have one set of CSS rules for when the screen is smaller than 800px and another set for larger screens.

If we wanted to follow a "mobile-first" approach, we could do the opposite; our base CSS could be focused on mobile devices, while our media query used the condition min-width: 800px to target wide-screen devices.

Media queries can also be used to query user preferences, such as a preference for reduced motion, or whether the user's device uses a dark theme:

@media (prefers-reduced-motion: reduce) {
    /* CSS rules for users who prefer reduced motion */}
@media (prefers-color-scheme: dark) {
    /* CSS rules for users who prefer a dark theme */}
